Overview:

Search bars are a staple of the digital experience. Whether you're looking for a specific video, tracking down a specific social media profile or looking for a perfect article, search bars help guide us to exactly what we're looking for. But while the core function is the same, each platform approaches this tool differently, and unfortunately not all websites do it well. As I dove deeper into the user experience of search bars. I realized a critical truth:

Search bars are only effective if they are usable and accessible

Without these qualities, even the most beautifully designed search tool will fail to reach it's full potential. I set out to explore this phenomenon across different applications, aiming to understand how inconsistencies in their designs might impact users. The objective was clear: identify common pain points and recommend a redesign that could enhance usability and accessiblity for all users.

More specifically, thoughout the duration of this case study I:
  1. Analyzed the input and output states of search bars across three applications
  2. Selected one of the three aforermentioned applications and created state models for both mouse and keyboard users
  3. Created a redesigned search bar that addresses some of the inconsistencies I noticed during the analyses
  4. Created a state model for the redesigned component

    Similarly to my input analyses, when observing the differences in search bar outputs across Google Chrome, TikTok, and Edstem, I realized that search bar implementation could benefit from the following standardizations:

    • Increasing the search bar size when it is in focus
    • Automatically populate relevant results as the user types
    • Outline the search bar in a noticeable color when it is in focus

    Conclusion

    The search bars I observed were most usable when they clearly showed the focus states, populated results as the user types and dimmed the background to help users concentrate. I incorporates these elements into my redesign to reduce distractions and make the search experience more intuitive. However, I initially, overlooked screen reader accessibility; to address this, I would add aria-labels and ensure that dynamic suggestions are announced and navigable. My redesign solves a mismatch by not assuming prior knowledge of the interface, unlike Chrome's current interface which can confuse first-time users with redundant icons and unclear interactions. Simplifying the interface and emphasizing the active state of the search bar helps users focus on input rather than the rest of the page's contents. Inaccessaible inputs like missing focus indicators or inconsistent keyboard navigation can significantly hinder users with visual or motor impairments. For exapmle, TikTok's interface is difficult to navigate with a keyboard but easy with a mouse, exemplifying how non-mouse users are often deprioritized in component design. Overall, mouse and touch users are prioritized, which can exclude users that rely on keyboards or assistive technologies.
